## 📁 文件保存机制

### 每天新增的文件（不会覆盖旧文件）

```
2026-01-24.html           ← 第1天简报
2026-01-25.html           ← 第2天简报
2026-01-26.html           ← 第3天简报
...
output/archive/2026-01-24.md
output/archive/2026-01-25.md
output/archive/2026-01-26.md
...
```

### 为什么不会被覆盖？

因为文件名包含**日期**，每天都是新的文件名：
- ✅ `2026-01-24.html` 只在1月24日创建
- ✅ `2026-01-25.html` 只在1月25日创建
- ✅ 两个文件完全独立，互不影响

## 🔍 GitHub Actions 工作流

### 每天自动运行

```yaml
- name: 生成简报
  # 生成新的 2026-01-25.html
  # 旧的 2026-01-24.html 不会被删除

- name: 提交到仓库
  run: |
    git add "$TODAY.html"           # 只添加新文件
    git add "output/archive/$TODAY.md"
    git add "output/archive/$TODAY.html"
    git add "index.html"             # 更新首页
    git commit -m "Auto: $TODAY 法律简报"
```

**关键点**：`git add` 只添加新文件，不会删除旧文件！
